# 案例1：客服 Chatbot（Prompt + 变量 + 记忆）

## 一、案例目标

- 搭建一个“电商客服助手”聊天应用。
- 让回答 **稳定、可控、可回归验证**：
  - 不确定就澄清
  - 超出规则范围就拒答/引导人工
  - 输出格式统一（要点 + 下一步）

验证数据：
- `data/faq_kb.md`
- `data/order_policy.md`
- `data/sample_queries.json`

---

## 二、Prompt 设计要点（建议结构）

### 1. 角色与目标

- 角色：电商客服助手
- 目标：基于已知规则回答售后/物流/发票/账号问题

### 2. 边界与拒答策略

- 不要编造政策；遇到规则缺失要说明“未找到明确规则”并建议转人工
- 不要输出敏感信息（如订单号、手机号等）

### 3. 输出格式（示例）

```
## 结论
- ...

## 依据
- ...

## 需要你确认
- ...
```

---

## 三、验证步骤（建议）

1. 用 `data/sample_queries.json` 里的“客服类问题”逐条提问。
2. 检查：
   - 回答是否与 `faq_kb.md` / `order_policy.md` 一致
   - 是否保持固定输出结构
   - 是否在缺信息时主动澄清
