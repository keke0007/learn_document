## Obsidian 使用指南（实战版）

### 1. Obsidian 的核心理念

- **本地 Markdown 为核心**：所有笔记本质上都是本地的 `.md` 文件，便于长期保存、迁移和用 Git/网盘同步。
- **双向链接与知识网络**：用 `[[页面名]]` 在笔记之间建立关联，自动形成知识图谱，而不是单纯的树状目录。
- **块级思维**：不仅可以链接整篇笔记，还可以链接到某一段/某一小节，实现「以块为单位」复用知识。

---

### 2. 高效使用技巧

#### 2.1 基本操作与推荐设置

- **推荐基础设置**
  - `设置 -> 编辑器`：开启「实时预览」、「自动配对括号/引号」。
  - `设置 -> 文件与链接`：新建笔记默认位置设为「当前文件夹」或「收件箱（Inbox）」。
  - `设置 -> 外观`：选择一个你顺眼的主题（深色/浅色），保证长时间阅读不累。

- **常用快捷键（Windows）**
  - `Ctrl + O`：快速打开笔记（全局搜索标题）。
  - `Ctrl + P`：命令面板，快速执行几乎所有操作。
  - `Ctrl + N`：新建笔记。
  - `Ctrl + E`：编辑/阅读模式切换（若未禁用）。
  - `Ctrl + Shift + F`：全库搜索。
  - `Ctrl + 点击链接`：在新标签页中打开笔记。

#### 2.2 链接与标签的最佳实践

- **页面链接**
  - 使用 `[[概念/主题]]` 建立知识之间的「显式连接」。
  - 对经常用到的知识（例如「MongoDB 索引原理」）单独建页，通过 `[[MongoDB 索引原理]]` 多次引用。

- **块级链接**
  - 输入 `[[页面名#小节标题]]` 链接到某个小节。
  - 在编辑模式下，选中一段文字 -> 右键 -> 复制块链接，即可得到 `^块ID`，实现到段落级别的精准链接。

- **标签（Tag）**
  - 用 `#领域/标签` 给笔记打标签，例如：`#数据库/mongodb`、`#学习/计划`。
  - 标签适用于「横切分类」（状态、主题），而非结构性分类（这种更适合用文件夹+链接）。

#### 2.3 日记与知识沉淀

- **日记/日志用法**
  - 利用「每日笔记」功能（或配合插件）记录：当天学习内容、问题清单、灵感碎片。
  - 每天回顾时，把有长期价值的内容整理到专门的主题笔记中，并加链接。

- **模板化记录**
  - 设计统一模板，例如「阅读笔记模板」、「学习案例模板」、「会议记录模板」，提高记录质量与速度。

### 3. 常用插件推荐（以学习与知识管理为主）

> 安装入口：`设置 -> 社区插件 -> 关闭安全模式 -> 浏览`，搜索插件名称安装。安装后记得在「已安装插件」中开启。

#### 3.1 基础增强类

- **Templater**
  - 用途：高级模板引擎，比内置模板更强大，可插入日期、动态内容、调用脚本。
  - 典型用法：
    - 一键生成「学习笔记模板」「会议记录模板」。
    - 在新建日记时自动生成日期、星期、待办栏等。

- **Advanced Tables**
  - 用途：增强 Markdown 表格编辑体验。
  - 功能：自动对齐、Tab 导航单元格、支持简单运算等。

- **Calendar / Periodic Notes**
  - 用途：提供日历视图 + 每日/每周/每月笔记模板。
  - 适合：做学习打卡、周复盘、月度总结。

#### 3.2 知识管理与查询类

- **Dataview**
  - 用途：把笔记当数据库用，支持用「查询语句」按条件自动生成列表。
  - 典型场景：
    - 列出所有带 `#学习/数据库` 标签的笔记，并按日期/难度排序。
    - 自动生成「待整理的学习笔记」列表。

- **Tasks**
  - 用途：增强待办管理，支持优先级、截止日期、查询等。
  - 使用方式：
    - 在任意笔记中用 `- [ ]` 记录任务，配合 `Tasks` 插件按标签、日期、状态聚合展示。

- **Tag Wrangler**
  - 用途：批量重命名、合并标签，避免标签系统混乱。
  - 使用时机：标签用了一段时间后，需要统一规范命名时。

#### 3.3 写作与阅读体验类

- **Minimal Theme / Blue Topaz 等社区主题**
  - 用途：改善外观与排版，提高阅读和写作舒适度。

- **Longform / Novel Word Count 等写作类插件**
  - 用途：适合长文写作、项目报告写作，支持章节管理、字数统计等。

#### 3.4 自动化与效率提升类

- **QuickAdd**
  - 用途：一键执行「创建笔记 + 应用模板 + 放入指定目录 + 打标签」等操作。
  - 典型场景：
    - 快速新建「学习案例笔记」，自动：
      - 使用指定模板
      - 存入 `学习/案例` 目录
      - 自动添加标签 `#学习/案例`

- **Hotkeys++ / Commander（若可用）**
  - 用途：扩展快捷键与命令，针对你的高频操作做键位绑定。

### 4. 适合大规模学习与技术笔记的结构建议

- **按「领域」划分文件夹**
  - 如：`数据库/`、`大数据/`、`语言与框架/`、`工具与实践/`。

- **按「主题」建立核心知识页面**
  - 例如：
    - `MongoDB 核心概念`
    - `MongoDB 索引设计`
    - `分布式系统一致性模型`

- **使用「索引页」串联知识**
  - 举例：`数据库学习索引` 中列出并链接到：
    - `[[MongoDB 核心概念]]`
    - `[[MongoDB 索引设计]]`
    - `[[MongoDB 副本集与分片]]`
    - `[[MySQL 与 MongoDB 对比]]`

- **用日记驱动知识沉淀**
  - 每天：
    - 在「每日笔记」中记下学到的点、遇到的问题。
    - 把重要内容提炼到对应的主题页，并用 `[[ ]]` 互相链接。

### 6. 总结：构建你的个人知识库

- **先从简单开始**
  - 不必一上来就设计完美的结构，先用「日记 + 少量主题页」起步。

- **用链接而非复制**
  - 当多个地方需要同一个概念时，优先用 `[[某概念]]` 链接到唯一的知识来源，避免信息分裂。

- **定期整理与回顾**
  - 每周花固定时间：
    - 回顾当周日记
    - 把零散内容整理成系统化的专题笔记
    - 清理和归并标签

- **让 Obsidian 成为「长期可进化」的知识系统**
  - 随着你在大数据、数据库、中间件等领域的深入学习，持续完善结构、模板和索引。
  - 目标不是「整理一次」，而是「让笔记系统陪你一起成长」。
